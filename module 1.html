Planning Laravel + Vue.js + Tailwind sur 2 mois

Semaine 1 – Laravel : Structure & Routing (Détails Journaliers)
  • Jour 1 : Installation Laravel, Composer, Artisan, configuration .env
  • Jour 2 : Structure MVC, routes (web.php), contrôleurs (artisan make:controller)
  • Jour 3 : Blade templates : layouts, includes, affichage dynamique
  • Jour 4 : Migrations, Seeders, Factories – Création et remplissage d'une base test
  • Jour 5 : Modèles Eloquent, relations hasOne, hasMany, belongsTo
  • Jour 6 : Création d'un CRUD complet : Articles ou Notes
  • Jour 7 : Refactorisation, organisation des fichiers, test de l'application
Semaine 2 – Authentification & Sécurité
  • Installation Laravel Breeze ou Jetstream
  • Login, inscription, vérification d'email
  • Middleware, rôles, redirections
  • Validation avancée, policies, gates
  • API sécurisée avec Sanctum (optionnel)
Semaine 3 – Vue.js : Composition API & Interactivité
  • Syntaxe script setup, ref, reactive, computed
  • Props, emits, events personnalisés
  • Composants, slots, logique réutilisable
  • Formulaires avec v-model et gestion d'erreurs
  • Projet test : interface de gestion des notes
Semaine 4 – Vue Router & Pinia
  • Création d'une SPA avec Vue Router
  • Navigation dynamique, pages protégées
  • Gestion de l'état avec Pinia
  • Stockage local, sessions, authentification frontend
Semaine 5 – Inertia.js et Intégration Laravel ↔ Vue
  • Installation d'Inertia.js avec Laravel + Vue
  • Pages dynamiques côté serveur
  • Transfert de données backend vers composants Vue
  • Formulaires Inertia et gestion d’erreurs
Semaine 6 – Tailwind CSS Avancé
  • Grille, flexbox, responsive
  • Dark mode, thèmes personnalisés
  • Animations, transitions avec Vue
  • Création de composants UI stylés
Semaine 7 – Projet Complet : Journal ou SmartCampus Mini
  • Laravel : structure, API, sécurisation
  • Vue : interface avec composants
  • Inertia : navigation fluide
  • Tailwind : design professionnel
Semaine 8 – Tests, Déploiement & Optimisation
  • Tests Laravel : Feature, Unit
  • Déploiement (Railway, Render ou autre)
  • Clean code, refactorisation
  • Ajout d’un README + documentation
